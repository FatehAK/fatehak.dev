VSCode's plugin ecosystem is vast, and there are tons of settings to configure as we go along our development work.

One fine day, I was adding a few extensions for working with [Astro](https://astro.build/) and was changing a configuration. At this moment I noticed what an unmaintainable mess the `settings.json` file has become after years of experimenting with various extensions, themes, custom configs and what not!

Sometimes there would be an extension config right in the middle of some default settings or the reverse. I grew tired of sifting through the `settings.json` file to understand where's what. Also, I felt we adjust our settings adhoc and forget to check if a group for it already exists.

## Implementation

To solve this, I thought let's quickly create a simple script to sort all settings by keeping the top level parent keys as the reference point.

```js {4,16}
const json = {}; // Pass content of settings.json here

function sortVSCodeSettings(json) {
  // STEP 1: group by parent keys
  const groups = {};
  for (key in json) {
    // if the key includes a `.` we consider it as a parent key to sort on
    if (key.includes('.')) {
      const parent = `$$${key.split('.')[0]}`;
      groups[parent] = { ...groups[parent], [key]: json[key] };
    } else {
      groups[key] = json[key];
    }
  }

  // STEP 2: flatten all groups
  let result = {};
  for (key in groups) {
    // a `$$` prefix means it's a parent key
    if (key.startsWith('$$')) {
      result = { ...result, ...groups[key] };
    } else {
      result[key] = groups[key];
    }
  }
  return result;
}

sortVSCodeSettings(json);
```

You have to pass the content of your `settings.json` as the input and you get a sorted json as the output.

> ;info;
> **Tip**: You can view your settings by typing in `Open User Settings (JSON)` in the command pallete.

My settings now feel super organized and accessible for sure!
